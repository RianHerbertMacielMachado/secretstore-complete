1

Criar conta no Railway

Acesse o Railway e crie sua conta gratuita

  1. Acesse https://railway.app
  2. Clique em "Login" → selecione "GitHub"
  3. Autorize o Railway a acessar sua conta GitHub
  4. Você terá $5 de crédito grátis/mês ou 500h no Hobby plan
ℹ️
Plano Trial (gratuito): $5 de crédito por mês. Para uso contínuo, upgrade para o Hobby ($5/mês) que inclui PostgreSQL e Workers suficientes para uma loja pequena.
2

Criar Projeto + Serviço PostgreSQL

Configure o banco de dados antes de conectar o app

  1. No dashboard Railway, clique em "New Project"
  2. Selecione "Provision PostgreSQL" (cria banco + projeto juntos)
  3. O Railway cria automaticamente um serviço PostgreSQL com as variáveis de conexão
  4. Clique no serviço PostgreSQL criado e vá na aba "Variables"
  5. Copie o valor de DATABASE_URL (vai ser usado depois)
✅
O DATABASE_URL no formato postgresql://postgres:senha@roundhouse.proxy.rlwy.net:PORT/railway será gerado automaticamente. Você não precisa criar nada manualmente — o Railway gerencia tudo.
3

Conectar Repositório GitHub

Adicionar o app Next.js ao projeto Railway

  1. No painel do projeto (que já tem o PostgreSQL), clique em "+ New Service"
  2. Selecione "GitHub Repo"
  3. Autorize o Railway a acessar seus repositórios se solicitado
  4. Selecione o repositório secretstore-complete
  5. Branch: main
  6. Railway detectará automaticamente o railway.json na raiz do projeto
⚠️
Não inicie o deploy ainda! Adicione todas as variáveis de ambiente primeiro (próximo passo). Um deploy sem as variáveis corretas vai falhar.

Verificar railway.json no Repositório

O arquivo já está no seu repositório GitHub

O arquivo railway.json já foi criado e enviado para o GitHub. Ele instrui o Railway como fazer o build e start da aplicação:

railway.json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS",
    "buildCommand": "npm install --legacy-peer-deps && npx prisma generate && npm run build"
  },
  "deploy": {
    "startCommand": "npx prisma migrate deploy && node .next/standalone/server.js",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 3
  }
}
🔍
O que cada comando faz:
• npx prisma generate — gera o Prisma Client com as tipagens
• npm run build — compila o Next.js com output: 'standalone'
• npx prisma migrate deploy — aplica as migrations no PostgreSQL (executa no start)
• node .next/standalone/server.js — inicia o servidor Next.js otimizado

Configurar Domínio Customizado (Opcional)

Use seu próprio domínio em vez do .railway.app

  1. No serviço Next.js, vá na aba "Settings"
  2. Em "Networking", clique em "+ Custom Domain"
  3. Digite seu domínio, ex: secretstore.com.br
  4. Railway mostrará o registro CNAME para configurar no seu DNS
  5. No painel do seu registrador de domínio (Registro.br, GoDaddy, etc.), adicione o CNAME
  6. Aguarde a propagação DNS (até 24h)
⚠️
Após configurar o domínio customizado, atualize as variáveis:
• NEXTAUTH_URL → https://seudominio.com.br
• NEXT_PUBLIC_SITE_URL → https://seudominio.com.br
Isso dispara um novo deploy automaticamente.

Pós-Deploy: Criar Admin e Dados Iniciais

Configure o primeiro usuário administrador

Com o banco vazio, crie o primeiro usuário admin diretamente pelo Railway CLI ou via interface:

Opção A — Railway CLI (recomendado)
# Instalar Railway CLI
npm install -g @railway/cli

# Autenticar
railway login

# Executar comando SQL no PostgreSQL do projeto
railway run npx prisma db seed

# Ou via psql direto:
railway connect PostgreSQL
Opção B — Pela interface web do Railway
# 1. Vá em seu serviço PostgreSQL no Railway
# 2. Clique na aba "Query" ou "Data"
# 3. Execute o SQL para criar o primeiro admin:

INSERT INTO "User" (id, name, email, password, role, "createdAt", "updatedAt")
VALUES (
  gen_random_uuid()::text,
  'Admin',
  'admin@seusite.com.br',
  -- Hash bcrypt da senha 'admin123' (troque após o primeiro login!)
  '$2b$10$rQZ8K1mN3pL7vX9wY2tU4eB6cD0oJ5sM8fH1nA3kE9qR7iG5aW4zO',
  'ADMIN',
  NOW(), NOW()
);
🔐
Segurança importante: Use o registro normal pelo site (rota /auth/register) e depois atualize o role para 'ADMIN' via SQL. Nunca coloque uma senha em texto plain no banco.

Problemas Comuns e Soluções

Erros frequentes no deploy Railway + Next.js

❌ Erro: "P1001: Can't reach database"
🔧
Verifique se a variável DATABASE_URL aponta para ${{Postgres.DATABASE_URL}} e não para um valor copiado manualmente. O Railway usa referências internas para comunicação entre serviços.
❌ Erro: "Cannot find module '.next/standalone/server.js'"
🔧
O build falhou antes de gerar o standalone. Verifique os logs de build. Certifique-se que o next.config.js tem output: 'standalone' descomentado.
❌ Erro: "Migration failed" no start
🔧
A migration tenta rodar antes do banco estar pronto. O Railway normalmente resolve isso com retry automático (restartPolicyType: "ON_FAILURE"). Se persistir, verifique se o serviço PostgreSQL está Running antes de fazer o deploy do app.
❌ Erro: "NEXTAUTH_URL mismatch"
🔧
O NEXTAUTH_URL deve ser exatamente a URL pública do app (com HTTPS e sem barra final). Se mudou o domínio, atualize a variável e faça redeploy. Também atualize nas configurações OAuth do Google/Discord.
⚠️ Fonts/imagens 404 em produção
🔧
O build standalone copia automaticamente os arquivos estáticos. Se ocorrer, verifique se os arquivos estão em public/ no repositório e se foram commitados.
Como verificar logs em produção via Railway CLI
# Instalar CLI
npm i -g @railway/cli

# Login e vincular projeto
railway login
railway link

# Ver logs em tempo real
railway logs

# Executar comando no ambiente de produção
railway run npx prisma studio
